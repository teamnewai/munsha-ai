# 09 — GLOBAL EXPANSION ARCHITECTURE ⭐
## النسخة الموسّعة (Expanded Blueprint)

| | |
|---|---|
| **الحالة** | 🟢 موسّع |
| **يبني على** | `organizations` (country/region) · `subscriptions` · RLS الجغرافي · جسر الامتثال |

> **ثمرة الصمود:** ما يُبقي الشجرة حيّةً عبر الحدود. الهدف: تمكين أي شركة في أي دولة من إدارة أعمالها عبر مُلكي — 195 دولة، وكل اللغات — **دون أن تتحوّل المنصة إلى وسيطٍ ماليّ أو مهنيّ غير مرخّص.**

## الفصل 1 — الفلسفة: نموّ بلا مخاطرة تنظيمية
مُلكي تُصدّر **البرمجيات**، لا الخدمات المهنية المنظَّمة. التوسّع العالمي يعني نشر طبقة التشغيل (المكتب الافتراضي + إدارة الأعمال) عالمياً، مع **جسر امتثالٍ ذكي** يحدّد ما يعبر الحدود بحرّية وما يحتاج وسيطاً محلياً مرخّصاً.

## الفصل 2 — جسر الامتثال الذكي (Compliance Bridge)
لكل زوج (دولة × مهنة) قاعدة:

| نوع الخدمة | عبر الحدود | المعالجة |
|-----------|-----------|----------|
| استشارة / تصميم / برمجة / تسويق / ترجمة | حرّ شبه كامل | تُفتح فوراً |
| محاماة / محاسبة / طب / هندسة معتمدة | مقيّد بترخيص | عبر وسيطٍ مرخّص محلياً |
| تحويل / حيازة أموال | مقيّد بشدّة | **خارج النطاق تماماً (مبدأ REOS)** |

```sql
compliance_rules(id, country, profession, cross_border_level, requires_local_broker, notes)
```

## الفصل 3 — خطة التوسّع المتدرّجة (Phased Rollout)
```text
السعودية (الأساس)
  → الخليج + مصر (تشابه تنظيمي ولغوي)
    → المملكة المتحدة (بوابة عالمية، إنجليزي)
      → الولايات المتحدة (سوق ضخم، تعقيد ولايات)
        → ماليزيا (بوابة آسيا الإسلامية)
          → الصين (بنية محلية منفصلة بالكامل)
```
كل سوق يتطلّب: ترجمة + أطر تنظيمية + بوابة دفع محلية + (أحياناً) كيان قانوني.

## الفصل 4 — التعدّد اللغوي (i18n)
- **مكتمل:** عربي (RTL أصلي) + إنجليزي.
- **بنية جاهزة:** ES / FR / DE / ZH / TR.
- **مبدأ:** لا ترجمة آلية لمنصّة B2B — الترجمة احترافية لأنّ المصطلحات التعاقدية/المالية حسّاسة.

## الفصل 5 — نظام العضوية العالمي
أرقام عضوية بصيغة `الدولة-المنطقة-التسلسل` (مثل `SA-RYD-001042`). العملات: تُعرَض بعملة المنشأة، وأسعار اشتراك مُلكي بالريال محوّلة استرشاداً.

## الفصل 6 — المدن الرقمية الموزّعة
يرتبط بمجلّد 07: كل دولة يمكن أن تمتلك **مدينتها الرقمية** (أحياء + أبراج + عناوين) ضمن شبكة مُلكي العالمية الواحدة.

## الفصل 7 — الأطر التنظيمية المرجعية
| النطاق | الأنظمة |
|-------|---------|
| السعودية | **PDPL** (خصوصية) · **REGA** (عقاري) · **ZATCA** (فوترة) · **نفاذ** (هوية) · **SAMA** (إن اقترب أيّ شيء من الوساطة المالية) |
| الخليج | أنظمة حماية البيانات والفوترة المحلية لكل دولة |
| عالمي | **GDPR** + المتطلبات الإقليمية |

## الفصل 8 — الصوت الجماعي (Collective Voice)
أداة تجمع مطالب المشتركين عبر الدول وتحوّلها مقترحاتٍ منظّمة لتطوير البيئة التنظيمية — تحوّل قاعدة المستخدمين إلى قوة دفعٍ تنظيمية.

## الفصل 9 — خارطة التنفيذ
1. ثبات السوق السعودي أولاً (Product-Market Fit محلي).
2. الخليج + مصر (نفس البنية، ترجمة + أطر محلية).
3. سوق إنجليزي واحد (UK) لإثبات النموذج عالمياً.
4. توسّع مدفوع بالطلب الفعلي، لا بالخريطة.

> **مبدأ حاكم:** لا توسّع لدولةٍ قبل وجود عملاء فيها يطلبونها — التوسّع نتيجة جذبٍ (pull)، لا دفعٍ (push).
