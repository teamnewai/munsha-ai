# CR-001 — Missing Authoritative Master Specification & Phase 4A/4B/4C Documents

- **Type:** Clarification / Documentation Correction (allowed in v1)
- **Status:** OPEN — awaiting authoritative documents
- **Raised:** 2026-06-09
- **Phase:** D1 — Foundation

## Summary

The D1 task brief names `THUL-NURAYN_v1_MASTER_SPECIFICATION.md` as the
**only source of truth** and requires implementing the PostgreSQL layer
"based on Phase 4A" and verifying "Phase 4A / 4B / 4C compliance." None of
these documents exist in the repository or its git history.

## Impact

Without the authoritative documents, the following could not be implemented
"exactly as specified" and were instead built from **conservative,
documented assumptions** derived from the task brief's own enumerations:

- Exact column definitions, types, nullability, and defaults per entity.
- Exact relationship cardinalities and `ON DELETE` behaviours.
- Exact index set (which columns, composite ordering, partial/expression).
- Exact partition keys, granularity, and partition count.
- Exact retention windows and archival mechanism.
- Exact **members** of each enumeration.

## Resolution Path

1. Supply the authoritative `THUL-NURAYN_v1_MASTER_SPECIFICATION.md` and
   Phase 4A/4B/4C documents.
2. Diff the D1 implementation against them.
3. Treat any divergence as a **bug fix** (allowed in v1) and reconcile.
4. Replace the derived placeholder spec in `docs/` with the authoritative file.
