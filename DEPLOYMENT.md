# دليل النشر — مُلكي (MULKI OS)

> هذا الدليل يجهّزك لنشر المنصة على **Vercel** وربطها بقاعدة بياناتك الحقيقية على **Supabase**.
> الكود **جاهز 100% للنشر**؛ يعمل بـ«وضع تجريبي» بدون مفاتيح، وببياناتك الحقيقية عند إضافتها.

---

## 1) النشر على Vercel (الطريقة الأسرع — 1-click)

1. ادخل [vercel.com](https://vercel.com) → **Add New → Project → Import Git Repository**.
2. اختر المستودع `teamnewai/munsha-ai`.
3. **Production Branch:** `claude/platform-build-scratch-u3w3wm` (أو ادمجه في `main` أولاً ليكون الإنتاج).
4. Framework: **Next.js** (يُكتشف تلقائياً) — لا تغيّر شيئاً.
5. أضِف متغيّرات البيئة (القسم 2) — أو انشر مباشرة بالوضع التجريبي ثم أضِفها لاحقاً.
6. **Deploy** → سيصدر رابطٌ حيّ خلال دقائق.

---

## 2) متغيّرات البيئة (Environment Variables)

أضِفها في **Vercel → Project → Settings → Environment Variables**:

| المتغيّر | القيمة | إلزامي؟ |
|---------|--------|---------|
| `NEXT_PUBLIC_SUPABASE_URL` | `https://fgincdqvhnuarqcdnsfe.supabase.co` | للبيانات الحقيقية |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | `sb_publishable_TGmirSYzNQDO8qL3PNf2Og_rYZh-K-E` | للبيانات الحقيقية |
| `NEXT_PUBLIC_APP_URL` | `https://mulki.sa` (أو رابط Vercel) | للـ SEO/Sitemap |
| `NEXT_PUBLIC_APP_NAME` | `مُلكي` | اختياري |
| `OPENAI_API_KEY` | مفتاحك | لوكلاء الذكاء (لاحقاً) |
| `STRIPE_SECRET_KEY` / `MOYASAR_SECRET_KEY` | مفاتيحك | للدفع (لاحقاً) |

> **المفتاح العام (anon/publishable) آمن** — مصمّم للظهور في المتصفح. لا تضع `service_role` في متغيّرات `NEXT_PUBLIC_*` أبداً.

بدون أول متغيّرين → يعمل الموقع في **الوضع التجريبي** (بيانات عرض). معهما + تسجيل الدخول → تظهر **بياناتك الحقيقية**.

---

## 3) قاعدة البيانات (Supabase)

- المشروع الحقيقي **موجود وجاهز**: `mulki-reos` (130+ جدول، 88 migration، بيانات حقيقية).
- **لا تُعد تشغيل أي migration** — المنصة الجديدة تقرأ من المخطّط الحالي **قراءة فقط** (لا تكتب ولا تعدّل).
- ملفات `supabase/migrations/0001_init.sql` و`0002_seed.sql` في المستودع هي **مرجعية** لنواةٍ نظيفة فقط — لا تُطبّق على `mulki-reos`.
- لتظهر البيانات الحقيقية: سجّل الدخول بحسابٍ له عضوية (`memberships`) في منشأة.

---

## 4) النطاق (Domain)

1. في Vercel → **Settings → Domains** → أضف `mulki.sa` (ووجّه DNS حسب تعليمات Vercel).
2. حدّث `NEXT_PUBLIC_APP_URL` للنطاق النهائي (يُحدّث robots/sitemap تلقائياً).

## 7) أوامر محلية

```bash
npm install
cp .env.example .env.local   # املأ المفاتيح (اختياري)
npm run dev                  # http://localhost:3000
npm run build && npm run start   # محاكاة الإنتاج
```
